- Java 객체 관리 & 멀티스레드

! 오늘의 배움: 데이터 바구니(Generic/Collection)와 열일하는 일꾼(Thread)
1. 포장 클래스 (Wrapper Class) & 제네릭 (Generic)
Why?: 자바의 기본형(int, char 등)은 가벼워서 좋지만, 컬렉션(바구니)에 넣거나 객체로 다뤄야 할 때 한계가 있음.
      그래서 얘네를 객체로 감싼 게 Wrapper Class(Integer, Character 등)임.

제네릭(<T>): 바구니에 '사과'만 담을지 '포도'만 담을지 미리 정하는 것.

ArrayList<Employee>: 이 리스트는 무조건 Employee와 그 자식들만 들어옴.

꺼낼 때마다 "너 Employee 맞니?" 하고 형변환(Casting) 안 해도 돼서 편하고 안전함.

2. 컬렉션 프레임워크 (Collection)
ArrayList: 가장 흔히 쓰는 바구니. 순서가 있고 중복을 허용함. 근데 중간에 있는 거 빼면 뒤에 애들이 앞으로 한 칸씩 당겨지느라 좀 힘들어함.

LinkedList: 데이터들이 앞뒤로 손을 잡고 있음. 중간에 끼워넣거나 빼는 건 손만 바꿔 잡으면 되니 빠른데, 특정 위치를 찾으려면 처음부터 쭉 가야 해서 검색은 좀 느림.

3. 스레드 (Thread) - 프로그램 안의 작은 일꾼
기본 개념: 자바 프로그램은 기본적으로 main 스레드 혼자 일함. 근데 동시에 여러 일을 하고 싶을 때 Thread 클래스를 상속받아 일꾼을 늘림.

스레드 생명 주기 (Life Cycle):

NEW: 일꾼 생성.

RUNNABLE: 일할 준비 완료 (CPU가 불러주길 기다림).

RUNNING: 실제 일하는 중 (run() 메소드 실행).

WAITING/BLOCKED: 잠깐 쉬거나(일시정지), 누가 화장실(공유 자원) 쓰고 있어서 기다리는 중.

TERMINATED: 일 다 끝남. 죽음.

4. 멀티스레드의 복병: 동기화 (Synchronization)
문제 발생: 여러 일꾼(Thread)이 하나의 식탁(Heap 영역의 공유 객체)에서 밥을 먹으면 싸움이 남. (데이터 꼬임)

해결책 (Lock): synchronized 키워드를 쓰면 한 일꾼이 식탁을 쓰는 동안 자물쇠를 걸어버림.

상태 제어 기술:

wait(): "난 지금 조건이 안 맞으니 **웨잇풀(Wait Pool)**에서 자면서 기다릴게."

notify(): "대기실에 있는 일꾼 하나 일어나서 일해!"

sleep(): "딱 정해진 시간 동안만 잠깐 졸다 올게."

5. 메모리 구조 다시 보기 (중요!)
Stack: 스레드마다 각자 가짐. 개인 수첩 같은 거라 다른 놈이 못 봄.

Heap: 모든 스레드가 공유함. 진짜 객체들이 사는 곳이라 여기서 싸움이 나기 때문에 Lock이 필요한 것임.

Java 핵심 개념 요약 정리 + 시험 오답 
1. 변수와 타입 (Data Types & Variables)
크기 비교: byte(1) < short(2) < int(4) < long(8) (단위: Byte)

변수 명명 규칙: 숫자로 시작할 수 없으며, 특수문자는 _와 $만 허용 됨.

Wrapper Class: 기본 타입(primitive)을 객체로 감싸는 클래스입니다.

대부분 앞글자를 대문자로 바꾸지만, char -> Character, **int -> Integer**는 형태가 다르니 주의

2. 메모리 영역과 객체 (Memory & Object)
Stack: 로컬 변수가 저장되며, 블록({})이 끝나면 즉시 소멸

Heap: new 연산자로 생성된 객체와 배열이 저장됩니다. 참조되지 않는 객체는 GC(Garbage Collector)가 알아서 치워줌 (개발자가 직접 정리하는 게 노노!)

String 특징: * length()는 메소드이므로 뒤에 괄호가 붙음. (s.length는 배열의 속성.)

String은 불변(Immutable) 객체입니다. substring()을 호출해도 원본 객체는 변하지 않고 새로운 문자열 객체 반환함.

3. 객체지향 프로그래밍 (OOP) & 다형성
다형성 (Polymorphism): * 부모 타입 변수에 자식 객체를 대입할 수 있습니다. (Parent p = new Child();)

Overloading: 이름은 같지만 파라미터가 다른 메소드를 여러 개 만드는 것.

Overriding: 부모의 메소드를 자식이 재정의하는 것. (생성자는 상속되지 않으므로 재정의할 수 없습니다.)

접근 제한자:

private: 클래스 내부에서만 접근 가능. (상속은 되지만 자식 클래스에서도 직접 접근은 불가)

default: 같은 패키지 내에서만 접근 가능. (키워드를 붙이지 않는 것이 기본값)

protected: 같은 패키지 + 상속받은 자식 클래스까지 허용.

Interface: 클래스는 단일 상속만 되지만, 인터페이스는 다중 상속이 가능하며 인터페이스끼리도 상속(extends)이 가능.

4. 예외 처리 (Exception)
NullPointerException: 객체가 생성되지 않은(null인) 참조 변수를 사용할 때 발생.

Checked vs Unchecked: * RuntimeException 계열은 컴파일러가 체크하지 않음.

사용자 정의 예외는 RuntimeException이나 Exception 중 아무거나 상속받아 만들 수 있음.

5. 컬렉션 & 최신 문법 (Collection & Lambda)
ArrayList vs Vector: 둘 다 배열 기반이지만, Vector는 멀티쓰레드 환경에서 안전(Thread-safe)하도록 설계.

ArrayList vs LinkedList: 중간 삭제/삽입이 빈번하면 LinkedList가, 인덱스 조회나 순차적 추가는 ArrayList가 유리합니다. (ArrayList는 중간 삭제 시 빈칸을 메우기 위해 데이터를 밀어냄.)

람다(Lambda): 함수형 인터페이스(메소드가 딱 하나인 인터페이스)의 구현체로 사용.

매개변수가 한 개일 때만 괄호 ()를 생략할 수 있음. (0개일 때는 생략 불가)
