# 📅 2026-04-08 - 상속과 toString 오버라이딩

## 1. 배운 개념

### 1) 상속 (Inheritance)
- 부모 클래스의 속성과 메소드를 자식 클래스가 물려받는 것
- 코드 재사용성과 유지보수성 향상

예:
- Employee → Manager, Engineer

---

### 2) 메소드 오버라이딩 (Overriding)
- 부모 클래스의 메소드를 자식 클래스에서 재정의
- 동일한 메소드 이름 사용

java

@Override
public String toString() {
    return super.toString() + " 취미:" + hobby;
}

3) super 키워드
부모 객체를 참조할 때 사용
부모의 메소드 호출 가능

-> super.toString() → 부모 정보 가져오기

4) toString()의 역할
객체를 문자열로 출력할 때 자동 호출됨

System.out.println(cust);
cust.toString()

cust.clone();
이건 그냥 쓰면 x

왜냐면:

Cloneable 인터페이스 구현해야 함
아니면 CloneNotSupportedException 발생

-> 지금 단계에서는 그냥 clone()은 나중에 배우는 개념

2. 문제 / 헷갈린 점
왜 toString을 재정의해야 하는지 이해 부족
super를 쓰지 않으면 어떻게 되는지 헷갈림

3. 해결 / 이해
기본 toString은 객체 주소값만 출력함
→ 사람이 보기 불편
그래서 오버라이딩해서:
→ 원하는 정보 출력하도록 변경
super를 사용하면:
→ 부모 정보 유지 + 추가 정보 확장 가능

4. 핵심 정리 (내 언어)
-상속은 기존 코드를 확장하는 구조
-toString은 객체 출력용 메소드
-super를 사용하면 부모 기능을 그대로 활용 가능

=> “상속 + 오버라이딩 + super = 객체지향 핵심 패턴”
