#  2026-04-08 - 객체와 메모리 구조

## 1. 배운 개념

### 1) 객체(Object)
- 객체 = 데이터(필드) + 메소드
- 현실 세계의 개념을 프로그램으로 표현한 것
- 객체 간의 관계를 통해 프로그램이 구성됨

---

### 2) 참조 타입 vs 기본 타입

- 기본 타입 (Primitive)
  - 값 자체를 저장 (int, double 등)
  - 스택에 저장

- 참조 타입 (Reference)
  - 객체의 주소(참조값)를 저장
  - 실제 데이터는 힙(Heap)에 저장
  - 배열, 클래스, 인터페이스 포함

---

### 3) 메모리 구조

- Stack: 변수 저장
- Heap: 객체 저장
- Method Area: 클래스 정보 저장

👉 객체는 Heap에 생성되고,  
👉 Stack에는 "참조값"만 저장됨

---

### 4) 객체 비교

- `==` : 주소(참조값) 비교
- `equals()` : 실제 내용 비교

---

### 5) 배열(Array)

- 같은 타입의 데이터를 연속된 공간에 저장
- 인덱스를 통해 접근

특징:
- 크기 변경 불가
- Heap 영역에 생성됨
- 기본값으로 초기화됨 (int → 0, boolean → false 등)

---

### 6) 객체지향 특징

- 캡슐화 (encapculation)
- 상속
- 다형성 ⭐ (가장 중요)

👉 유지보수와 재사용성을 높이기 위한 구조

---

## 2. 헷갈린 점 / 문제

- 참조값과 실제 주소의 차이가 헷갈림
- 왜 자바는 포인터를 직접 사용하지 않는지 이해 부족

---

## 3. 해결 / 이해한 내용

- 자바는 포인터 대신 참조값을 사용하여
  → 메모리 안정성과 관리 효율을 높임

- 객체의 실제 주소는 변경될 수 있기 때문에  
  → 참조값(해시 기반)으로 접근하는 구조

---

## 4. 핵심 정리 (내 언어)

- 객체는 Heap에 저장되고, 변수는 그 주소를 참조한다
- 자바는 직접 주소를 다루지 않고 참조값으로 관리한다
- 객체지향은 유지보수를 쉽게 하기 위한 구조이다
