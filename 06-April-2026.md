# 📅 2026-04-06 - JAVA 기초 & JVM 메모리구조

## 1. 배운 개념
1) compile vs interpreter
-컴파일 (c,c++) : 실행 전 미리 번역, 속도 빠름
-인터프리터 (JS) : 실행 시 실시간 번역, 컴파일 보다 느림.
-> JAVA : 두방식 모두 사용. 컴파일러가 .java -> .class(바이트코드)로 바꾸고, JVM 에서 인터프리터&JIT컴파일러가 실행.

2) JVM(Java Virtual Machine) 탄생배경
- 플랫폼 독립성 :  과거 임베디드 시스템 -> 기기(os)마다 컴파일, 자바는 JVM 설치되면 어디서든 동일한 코드 실행가능.
- JIT(Just-In-Time)컴파일러 : *초기 인터프리팅 방식 (느림), 반복되는 코드를 JIT가 네이티브 코드로 컴파일해둠.
※ 처음실행 느릴 수 있음, 갈수록 빨라짐.

3) JVM 메모리 구조와 GC (Garbage Collector)
자바 != C/C++ 개발자가 메모리 직접 해체 x , GC가 자동으로 관리.
-> Memory 영역
ㆍstack : 기본 타입 변수 및 메소드 호출 정보 저장.
ㆍHeap : 참조타입(객체)이 저장되는공간.
  ㆍ Eden : 새로 생성된 객체, 꽉차면 Minor GC 발생.
  ㆍ survivor : Eden에서 살아남은 객체 이동.
  ㆍ Old Generation : 오랫동안 살아남은 객체 저장. 꽉차면 Major GC(Full GC)발생.
ㆍNon-Heap(Permanent/Metaspace) : 클래스 메타데이터, 스테틱(static)정보, 캐시 등이 저장되는 공간
=> GC알고리즘 (Key word)
ㆍStop-the-world : GC실행을 위해 JVM이 애플리케이션 실행 멈추는 현상.
ㆍG1 GC : 최근 가장 많이 쓰이는 효율적인 알고리즘 (모바일/서버 영역에서 중요)

6) Java 연산자, 제어문, 메서드
1. 연산자 (Operators)
  ㆍ 증감 연산자 (++,--)
      - 속도 : [i=i+1 < i++] (변수 찾는 과정 간소화됨.)
      - 전위 연산 (++i) : 값을 먼저 증가 -> 다른연산 수행 [우선순위 높음.]
      - 후위 연산 (i++) : 다른연산 수행 -> 값을 증가 [우선순위 낮음.]
  
  ㆍ 산술 연산자 ※ 주의 사항 
      - 정수 나눗셈 : 5/2 = 2(소숫점 버림). 실수 얻으려면 5/2.0 처럼 실수 연산 해야함.
      - 나머지 연산(%) : 나눈 후 남은 값을 구함.
      - * 자바만의 특징 ( 실수연산 ) :
              - 5/0.0 -> infinity (무한대)
              - 5%0.0 -> NaN (Not a Number,숫자가 아님)
              - 정수를 0으로 나누면 ArithmeticException 예외 발생.
  ㆍ 오버플로우 & 언더플로우
      - 오버플로우 : 타입이 허용하는 최대값을 넘칠 때. (ex. 127+1 -> -128)
      - 언더플로우 : 최솟값보다 작아질 때.
 2. 제어문(Control Flow)
   ㆍ 조건문 : if vs switch
       - if 문 : 범위 연산 (>=, &&,||)이 가능하여 유연하지만, 조건이 너무 많으면 속도가 느려질 수 있음.
       - switch 문 : 변수값과 일치하는 case로 직접 점프하므로 병렬 처리에 유리하고 빠름.
               ㆍ == 비교만 가능(범위 비교 불가).
               ㆍ break가 없으면 다음 case를 연달아 실행하므로 주의.
               ㆍ 자바 7부터는 String 타입도 사용 가능.
  ㆍ 반복문 : for vs while
         - for 문 : 반복 횟수를 정확히 알 때 사용. (for(초기화; 조건식; 증감식))
         - while 문 : 조건에 따라 반복할 때 사용. (ex : 네트워크 데이터 수신 등 횟수가 불명확할 때)
         - do-while 문 : 조건과 상관없이 최소 1번은 무조건 실행한뒤 조건을 확인.
  ㆍ  : break vs continue
         - break : 가장 가까운 반복문을 즉시 종료하고 탈출.
         - continue : 아래 코드 무시하고 다음 반복 (증감/조건식) 으로 넘어감.( 특정 조건 건너뛸때 유용)
7) 메서드 (Mehod)- "재사용 가능한 코드 뭉치"
- 미리 만들어둔 자판기 or 레시피 
- 매번 코드를 복+붙 -> 메모리 낭비, 수정 힘듦 => 이름 붙여서 따로 빼놓는것
 구조 : 
 [제한자] [리턴타입] [메서드명] ([매개변수/인자]) {
 //실행할 코드 
 return [결과값];
 }
         - 리턴타임 : 자판기에서 나올 결과물의 종류(숫자면int, 없으면 void)
         - 매개변수 (인자) : 자판기에 넣는 동전이나 재료.
         - void : "난 실행만 하고 돌려줄 결과물은 없어! 라는뜻
         - static : static 메서드(ex. main) 안에서는 static이 붙은 것들만 바로 호출 가능
