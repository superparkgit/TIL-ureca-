#  TIL - Java 객체지향 핵심 개념 정리

## 1. 메서드 오버라이딩 (Method Overriding)

오늘은 상속 관계에서의 메서드 재정의 개념을 학습했다.
오버라이딩은 부모 클래스의 메서드를 자식 클래스에서 동일한 시그니처(이름 + 매개변수)로 다시 정의하는 것이다.

* 이름과 매개변수는 반드시 동일해야 한다.
* 리턴 타입

  * primitive 타입은 반드시 동일해야 한다.
  * reference 타입은 동일하거나 하위 타입 반환 가능하다.
* 접근 제어자는 더 넓은 범위로만 변경 가능하다.
* 예외는 동일하거나 더 작은 범위로 선언 가능하다.

 이를 통해 기존 코드를 수정하지 않고 기능을 변경할 수 있어 유지보수성이 높아진다.

---

## 2. 접근 제어자 & 사용 제어자

### 접근 제어자 (Access Modifier)

* public: 어디서든 접근 가능
* protected: 상속 관계 + 같은 패키지
* default: 같은 패키지
* private: 해당 클래스 내부에서만 접근 가능

 데이터 보호 및 접근 제한을 위한 핵심 개념이다.

### 사용 제어자 (Usage Modifier)

* static: 객체 생성 없이 사용 가능 (클래스 단위)
* final: 수정 불가 (변수), 오버라이딩 금지 (메서드), 상속 금지 (클래스)
* abstract: 구현되지 않은 메서드

---

## 3. 캡슐화 (Encapsulation)

객체의 데이터를 보호하기 위해 변수는 private으로 선언하고,
외부에서는 getter/setter를 통해 접근하도록 하는 방식이다.

 목적

1. 데이터 보호
2. 내부 구현 은닉 → 유지보수성 향상

---

## 4. 상속 (Inheritance)

상속은 기존 클래스를 재사용하면서 기능을 확장하는 개념이다.

* 단순 객체 생성: 재사용만 가능
* 상속: 재사용 + 기능 변경(오버라이딩)

 코드 중복을 줄이고 확장성을 높인다.

---

## 5. JVM 메모리 구조

객체 생성 시 메모리 구조에 대해 이해했다.

* Stack: 지역 변수 및 참조 변수 저장
* Heap: 객체 생성 영역
* MetaSpace: 클래스 정보 저장

 객체는 Heap에 생성되고, 참조 변수는 Stack에 저장된다.

## 1. 다형성 (Polymorphism) 핵심 개념

다형성은 하나의 타입으로 여러 객체를 다룰 수 있는 성질이다.
즉, **코드는 동일하지만 실행 결과는 달라진다.**

```java
A a = new B(); // 업캐스팅
```

* 변수 타입: A
* 실제 객체: B

 **메서드는 실제 객체(B)의 메서드가 실행된다.**

## 2. 타입에 따른 접근 가능 범위

```java
class A {
    void m1() {}
    void m2() {}
}

class B extends A {
    void m1() {}  // 오버라이딩
    void m3() {}  // 새 메서드
}

A a = new B();
```

 호출 가능 여부

* a.m1() → o (B의 m1 실행)
* a.m2() → o (A의 m2 실행)
* a.m3() → x (A 타입에는 없음)

 정리

> **"변수 타입이 접근 범위를 결정한다"**

---

## 3. 메서드 실행 기준 (매우 중요)

 결론 한 줄

> **"실행은 객체 기준, 접근은 타입 기준"**

| 구분              | 기준     |
| --------------- | ------ |
| 변수/메서드 접근 가능 여부 | 왼쪽 타입  |
| 실제 실행되는 메서드     | 오른쪽 객체 |

## 4. 업캐스팅 vs 다운캐스팅

### 업캐스팅 (자동)

```java
A a = new B();
```

* 안전함
* 다형성의 핵심

---

### 다운캐스팅 (강제)

```java
B b = (B) a;
```

* 실제 객체가 B일 때만 가능
* 아니면 x ClassCastException

---

## 5. 오버로딩 vs 오버라이딩

| 구분 | 오버로딩        | 오버라이딩   |
| -- | ----------- | ------- |
| 기준 | 파라미터        | 상속      |
| 시점 | 컴파일         | 실행      |
| 특징 | 이름 같고 인자 다름 | 시그니처 동일 |

## 6. 메서드 시그니처

 구성

* 메서드 이름
* 매개변수

 포함 안됨

* 리턴 타입

 오버라이딩 판단 기준

## 7. 상속 구조에서의 메서드 호출

### 구조

```
A ← B ← C ← D
```

---

### 케이스 분석

```java
A a = new B();
```

* a.ma() → B의 ma() 실행 (오버라이딩)
* a.mb() → x (없으면 컴파일 에러)
* a.mb(3) → A의 mb(int) 실행

---

```java
A a = new C();
```

* a.ma() → C의 ma()
* a.mb(3) → C의 mb(int)

---

```java
B b = new D();
```

* b.mc() → x (B에 없음)
* b.ma() → C의 ma() (상속받은 것)
* b.mb() → D의 mb()
* b.mb(3) → D의 mb(int)

## 9. 다형성의 진짜 목적

 핵심

* 하나의 타입으로 모든 자식 관리

```java
A[] arr = new A[10];
arr[0] = new B();
arr[1] = new C();
arr[2] = new D();
```

 장점

* 코드 통합
* 유지보수 쉬움
* 확장성 ↑

## 10. 인터페이스까지 가야 완성되는 이유

```java
인터페이스 i = new D();
```

 의미

* 구현체(D)는 바뀌어도
* 사용하는 코드는 안 바뀜

 핵심 효과

* 결합도 낮춤 (decoupling)
* 확장성 극대화

##  핵심 한줄 정리

* 다형성 = "부모 타입으로 자식 객체를 다룬다"
* 접근 = 타입 기준
* 실행 = 객체 기준
* 유지보수와 확장을 위한 필수 개념
* 인터페이스까지 가야 진짜 완성

## 7. 업캐스팅 & 다운캐스팅

### 업캐스팅 (Upcasting)

* 자식 → 부모 자동 변환
* 다형성의 핵심

### 다운캐스팅 (Downcasting)

* 부모 → 자식 강제 변환
* 실제 객체 타입과 다르면 예외 발생

 다운캐스팅은 필요한 경우에만 사용하는 것이 좋다.

---

## 8. 동적 바인딩 (Dynamic Binding)

자바는 실행 시점에 어떤 메서드를 호출할지 결정한다.

 오버라이딩된 메서드는 항상 실제 객체 기준으로 실행된다.

---

## 9. static

* 클래스 로딩 시 1번 실행
* 객체 없이 사용 가능
* 공용 기능 구현에 적합

---

## 10. final

* 변수: 값 변경 불가
* 메서드: 오버라이딩 금지
* 클래스: 상속 금지

---

## 11. Object 클래스

모든 클래스의 최상위 클래스이며,
모든 객체는 Object 타입으로 참조될 수 있다.

오늘 학습을 통해 객체지향의 핵심은 단순한 문법이 아니라
유지보수성과 확장성을 고려한 설계라는 것을 이해했다.

특히 다형성을 활용하면 코드 중복을 줄이고,

앞으로는 단순 구현보다 구조를 먼저 고민하는 습관을 가져야겠다고 느꼈다.
